product name

What we have here is a website that allows users to create and customize a storefront where they can sell products of their choice. In particular, our store creation page allows them to build their store out of individual components without touching a line of code.

To view our website, start from index.html and follow the links from there. As we are not running on a server yet, logging in and signing up is not yet implemented. The store creation page is accessible and functional, but actually saving/generating the store does not work yet.

Because of this, we have prepared an example of what a generated store would look like in the Examples tab. Note that once the example store is entered, there are no links back to the main website - this is an intentional design choice, to keep shoppers concentrated on the goods themselves.

Navigation for A1

Front page of the product website: index.html
This includes the account creation and login functionality, which will be inplemented for the second assignment.

Examples page: examples.html
This page will include links to user's pages, if they choose to opt-in to the page.

About us: aboutus.html
The about us page, including pictures and bios/roles. Linked to from the navbar as shown in index.html

Store Page settings page: settings.html
This is the page that allows the user to modify their store page. Linked from index.html in the nacbar under the Create Store! button, and is the page the user will be sent to upon login.

Example store page: example_store_*.html
These pages show a page as it could be created using our product, albeit roughly customized.
Subpages:

  • example_store_home.html: an optional splash page
  • example_store_products.html: this showcases the Text Block and Product List elements from the settings page
  • example_store_searchresults.html: the page that the a consumer is directed to after querying a search using the search bar.

General layout

The root of the webpage is index.html. It includes a short introduction explaining the purpose of the site (easy online storefront) and includes an account creation form. In the navbar at the top, there are links to the about us page, the list of example store made using the product (to be filled for the second assignment, once account/page creation is implemented), and the Create Store! button, which goes to the page for store customization (settings.html) if the user is logged in (not yet implemented).

The various preview buttons in settings.html will open a preview tab/window showing current changes.

Store pages are entirely separate from our main page, barring the link back in the footer, unless the user opts-in to being linked from the examples page. The stores are separate entities, just created with our product.

Final Navigation

Once the server is implemented, index.html will be the root, whose working title is easystorefront.com. Any stoe page will use the user's store name as the subdirectory (i.e. easystorefront.com/storename) and will have subdirectories for home, search, etc. (i.e. easystorefront.com/storename/search).
