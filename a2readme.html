Features

Server Integration

Account Creation and Login

The Create Account section on the front page sends a request to the server to cerate an account. If there is no conflict in the fields (i.e. the email is not in use) then the user's information is added to the database, allowing login using those credentials.

The login itself checks against existing user accounts and if such an account existssets up a cookie that notifies the rest of the site of the login. As of now, the cookie stores the user's email address in plain text, but this will be changed in assignment 3.

Store Edit Page

After login, the user has access to the create store page with which they can customise the look and content of their store. The values are stored to the server on save. Every tab other than the images save the full set of elements, while the image tab requries each image to be saved individually, due to the existance of the image file present on the server which requires more explicit adding and deletion. On page load, the user's existing settings are loaded into the fields. The drop-downs for image are populated by existing images in the user's image tab.

Page generation

When the edit page is saved, on any of the pages, page respecting the changes are generated.

General page features

The contact us page on the store page shows the information saved in the user's settings. Carousel image display. Dynamic search.

Note that while the generated stores are for the most part fully functionally, the ability to actually spend money on the products is not included. This has been considered outside the scope of the project since the very beginning, as our main features are the store customization/generation and its interface.
